# IKUMEE（イクミー）アプリ仕様書

**文書版数:** 1.0  
**作成日:** 2026年7月23日  
**対象:** アプリ提案・操作デモ用静的Webプロトタイプ

## 1. 文書の目的

本書は、移動支援マッチングアプリ「IKUMEE」の目的、対象利用者、機能、画面、操作フロー、安全方針および技術構成を定義する。今回の成果物は提案内容を動画で説明するための静的プロトタイプであり、実際のマッチング、決済、本人確認、位置情報通信は行わない。

## 2. アプリ概要

IKUMEEは、「移動や日常生活で少し手助けが必要な人」と「近くにいて少し時間が空いている人」をリアルタイムにつなぐ地域支援アプリである。

依頼者は、道案内、荷物運び、付き添い、駅員への声かけ、受付補助など、短時間で解決できる困りごとを投稿する。サポーターは近くの依頼から自分にできるものを選んで引き受け、支援完了後に依頼者が設定した報酬を受け取る。

**コンセプト:** 「助けてほしい」と「手伝いたい」をつなぎ、誰もが安心して移動できる地域をつくる。  
**キャッチコピー:** 移動に、やさしい出会いを。

## 3. 背景と解決する課題

一般的な地図アプリでは目的地までの経路を確認できるが、利用者の身体状況に応じた坂道、段差、路面状態、混雑、エレベーターの位置などを十分に判断できない場合がある。バスやタクシーにも、乗り場までの移動、待ち時間、降車後の移動、料金といった課題がある。

IKUMEEは、地図情報だけでは解決しにくい「人による短時間の支援」を地域内で迅速に提供する。

## 4. 対象利用者

### 4.1 依頼者

- 妊婦
- 車いすユーザー
- 高齢者、要介護者
- 足の不自由な人
- 子連れ、ベビーカー利用者
- 外国人
- けが人

### 4.2 サポーター

- 近隣にいて短時間の空き時間がある人
- 道案内、荷物運び、付き添いなどを安全に行える人
- 本人確認と利用ルールへの同意を完了した人を想定する

## 5. 想定移動シナリオ

基本シナリオは「駅から病院までの移動」とする。

1. 駅：案内表示や目的の設備が見つからない。
2. 切符売り場：券売機の操作方法や購入方法が分からない。
3. ホーム：エレベーター、スロープ、乗車位置、駅員への依頼方法が分からない。
4. 歩道橋：階段が妊婦、けが人、ベビーカー利用者の負担になる。
5. 道：坂道、段差、狭い歩道、混雑、点字ブロックなどにより通行しづらい。
6. 入口・受付：病院入口や受付の位置、受付手続きが分からない。
7. 病院：目的地への到着と支援完了を確認する。

操作デモでは、ベビーカーと荷物がある子連れの妊婦が、駅から病院までの付き添いと荷物運びを依頼する。

## 6. サービス範囲

### 6.1 対象とする支援

- 道案内、安全な経路の提案
- 手荷物の運搬
- 徒歩移動の付き添い
- 駅員や施設スタッフへの声かけ
- 券売機など公共設備の操作案内
- 病院や施設の入口・受付までの案内
- 受付手続きの補助

### 6.2 対象外とする支援

- 利用者本人の抱き上げ、移乗介助
- 医療行為、服薬補助、専門的な介護行為
- 自家用車への同乗、送迎
- 法律または施設規則に反する行為
- サポーターの能力を超える危険な作業

## 8. 機能仕様

### 8.1 共通機能

- デモアカウントによるログイン
- ログイン画面へのデモID・パスワードの自動入力
- パスワード表示・非表示切り替え
- ログアウト
- 依頼者の初回設定として年齢層と表示モードを選択
- 本人確認済み表示
- 評価表示
- 通報・緊急連絡・位置共有の疑似表示
- マッチング相手との文章・画像チャットの疑似送信
- スマートフォンを中心としたレスポンシブ表示

### 8.2 依頼者機能

| ID | 機能 | 内容 |
|---|---|---|
| R-01 | 依頼作成 | 支援内容、出発地、目的地、日時、利用者の状況、詳細を入力する。 |
| R-02 | 報酬設定 | 依頼者が500円、800円、1,000円などから報酬を設定する。 |
| R-03 | 内容確認 | ルート、日時、支援内容、所要時間、報酬を確認する。 |
| R-04 | サポーター募集 | 近隣のサポーターを検索している状態を表示する。 |
| R-05 | マッチング確認 | サポーターの氏名、評価、本人確認、経験、到着予定を表示する。 |
| R-06 | 合流・支援開始 | 合流後、位置共有を伴う支援中画面へ移動する。 |
| R-07 | 支援完了 | 病院到着後、支援完了を確定する。 |
| R-08 | 評価・お礼 | 星評価、メッセージ、支払う報酬を確認する。 |
| R-09 | かんたんモード | 大きな文字・太字・大きなボタンを使用し、機能を簡略化する。10代または60歳以上では初期選択状態とする。 |
| R-10 | かんたん依頼 | 現在地、現在時刻、高齢者という状況、依頼文を自動設定し、目的地だけで募集できる。 |
| R-11 | キャンセル | 募集中から待ち合わせ前まで、理由を選択して依頼をキャンセルできる。 |
| R-12 | チャット | 通常モードとかんたんモードで、文章または端末内の画像を会話欄へ追加する。実際の送信は行わない。 |

### 8.3 サポーター機能

| ID | 機能 | 内容 |
|---|---|---|
| S-01 | 受付状態表示 | 「手伝えます」の受付状態を表示する。 |
| S-02 | 近隣依頼一覧 | 距離、支援内容、対象者、所要時間、報酬を一覧表示する。 |
| S-03 | 依頼詳細確認 | 依頼内容、集合場所、目的地、依頼者情報、安全注意を確認する。 |
| S-04 | 依頼引き受け | 選択した依頼を引き受け、依頼者とマッチングする。 |
| S-05 | 合流 | 集合場所、依頼者のプロフィール、連絡手段を表示する。 |
| S-06 | 移動支援 | 推奨ルートと各地点での支援内容を表示する。 |
| S-07 | 支援完了 | 活動時間と報酬を表示する。 |
| S-08 | 依頼マップ | 現在地周辺の依頼を地図形式で確認する。 |
| S-09 | 活動履歴 | 過去の支援内容、日付、活動時間、報酬を確認する。 |
| S-10 | 辞退 | 待ち合わせ前まで、理由を選択して引き受けた依頼を辞退できる。 |
| S-11 | チャット | 依頼者との文章・画像チャットを疑似体験する。実際の送信は行わない。 |

## 10. 操作フロー

### 10.1 依頼者フロー

ログイン → 依頼作成 → 内容確認 → 募集 → マッチング → 合流 → 支援開始 → 病院到着 → 評価・お礼

かんたんモードでは、ログイン → 任意の年齢選択 → 目的地入力 → 募集 → 合流 → 支援開始 → 到着 → かんたんホーム、の最小操作とする。完了後もモードを維持する。

### 10.2 サポーターフロー

ログイン → 近隣依頼を閲覧 → 詳細確認 → 引き受け → 合流 → 支援開始 → 推奨ルートで移動 → 支援完了 → 報酬確認

## 11. 安全・信頼性仕様

- 電話番号および本人確認済みであることを画面上に表示する想定とする。
- 依頼者とサポーターの相互評価を導入する想定とする。
- マッチング中だけ相手と現在地を共有する想定とする。
- 通報、運営への相談、緊急連絡への導線を表示する。
- 依頼内容に対象外行為が含まれないことを確認する。
- 危険を感じた場合は支援を中断し、駅員、施設職員、運営、緊急機関へ連絡する。
- 実サービス化では、個人情報保護方針、利用規約、保険、事故対応、報酬決済、未成年者の利用条件を別途定義する。

## 12. デザイン仕様

- 白と青を基本色とする。
- タイミーのような親しみやすく、操作対象が明確なカード型UIを採用する。
- 重要な操作は青色の大きなボタンで表示する。
- 角丸、適度な余白、短い文章により心理的負担を減らす。
- アイコンと文字を併用し、文字だけに依存しない。
- スマートフォン表示を優先し、PCでは最大幅480pxのアプリ表示とする。
- 小画面では画面幅いっぱいに表示する。

主なデモデータは次のとおりである。

- 依頼者：佐藤さくら、子連れの妊婦、本人確認済み
- サポーター：山田健太、評価4.9、移動サポート経験あり
- ルート：桜ヶ丘駅中央改札からみらい総合病院受付

## 15. 非機能要件

- スマートフォン幅320px以上で主要操作が行えること。
- PCでも画面が崩れず閲覧できること。
- ボタンや入力欄が指で操作しやすい大きさであること。
- 主要画面の文章が日本語で読みやすいこと。
- GitHub PagesのHTTPS配信で動作すること。
- JavaScriptが有効な一般的な最新版ブラウザで動作すること。
- 実在する個人情報をデモ画面に表示しないこと。

## 16. 受け入れ条件

- 入口から依頼者用・サポーター用の各URLへ移動できる。
- 両方の画面で指定デモアカウントによりログインできる。
- 不正なIDまたはパスワードではエラーを表示する。
- 依頼者側で、依頼作成から評価までの流れを確認できる。
- サポーター側で、依頼選択から報酬確認までの流れを確認できる。
- 本人確認、評価、通報、緊急連絡、位置共有を確認できる。
- 対象外となる危険な支援が画面上で案内される。
- スマートフォン表示で横方向にはみ出さない。
- Node.jsやバックエンドなしでGitHub Pages上に配置できる。

## 17. 今回実装しない機能

- 実際のユーザー登録・本人確認
- 実際のGPS取得、地図、経路探索
- リアルタイム通知・マッチング
- チャット、通話
- オンライン決済、報酬振込
- データ保存、履歴同期
- 運営者用管理画面
- 多言語翻訳

これらは画面上で利用イメージのみを示す。

## 20. 注意事項

本プロトタイプは授業での提案・紹介動画撮影を目的とする。実サービスとして公開・運用する場合は、認証情報をJavaScript内に置かず、安全なバックエンド認証を導入する必要がある。また、個人情報保護、決済、事故、保険、資格が必要な支援、地域法令などについて専門家を含めた検討が必要である。
